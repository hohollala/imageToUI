📋 개요

SSH MCP Server는 AI 클라이언트(Claude, Gemini 등)가 원격 서버에 SSH 연결을 통해 명령어를 실행하고 파일을 조작할 수 있도록 해주는 Model Context Protocol (MCP) 서버입니다.

🔗 SSH 연결 관리

다중 SSH 연결을 안전하게 관리하고 재사용할 수 있습니다.

⚡ 명령어 실행

원격 서버에서 명령어를 실행하고 결과를 실시간으로 받을 수 있습니다.

📁 파일 조작

원격 서버의 파일을 읽고, 쓰고, 목록을 확인할 수 있습니다.

🔐 보안 인증

비밀번호, SSH 키, SSH 에이전트 등 다양한 인증 방식을 지원합니다.

🛡️ 에러 처리

연결 실패, 명령어 오류 등 다양한 상황에 대한 안전한 에러 처리를 제공합니다.

📊 모니터링

연결 상태, 명령어 실행 시간 등 상세한 모니터링 정보를 제공합니다.

SSH 연결 설정 파일 (.sshenv)

ssh_init 도구를 사용하여 .sshenv 파일을 생성하면 SSH 연결 정보를 안전하게 관리할 수 있습니다. JSON 형식을 지원합니다:

JSON 형식 (.sshenv)

{ "servers": { "development": { "DEV_HOST": "192.168.1.100", "DEV_USER": "ubuntu", "DEV_PASSWORD": "your_password", "DEV_KEY_PATH": "~/.ssh/id_rsa", "DEV_PORT": 22 }, "staging": { "STAGING_HOST": "staging.example.com", "STAGING_USER": "deploy", "STAGING_KEY_PATH": "~/.ssh/staging_key", "STAGING_PORT": 22 }, "production": { "PROD_HOST": "prod.example.com", "PROD_USER": "admin", "PROD_KEY_PATH": "~/.ssh/prod_key", "PROD_PORT": 22 } }, "defaults": { "DEFAULT_PORT": 22, "DEFAULT_TIMEOUT": 30 }, "templates": { "web_server": { "TEMPLATE_USER": "www-data", "TEMPLATE_PORT": 22 } } }

기존 KEY=VALUE 형식도 지원

# 개발 서버 DEV_HOST=192.168.1.100 DEV_USER=ubuntu DEV_PASSWORD=your_password DEV_KEY_PATH=~/.ssh/id_rsa # 운영 서버 PROD_HOST=prod.example.com PROD_USER=admin PROD_KEY_PATH=~/.ssh/prod_key

환경 변수 설정 파일 (.env)

프로젝트 루트에 .env 파일을 생성하여 다음 설정을 추가할 수 있습니다:

# SSH MCP Server Configuration # Server Configuration SSH_MCP_DEBUG=false SSH_MCP_MAX_CONNECTIONS=10 SSH_MCP_TIMEOUT=30 SSH_MCP_LOG_LEVEL=INFO # MCP Protocol Configuration SSH_MCP_PROTOCOL_VERSION=2024-11-05 SSH_MCP_SERVER_NAME=ssh-mcp-server SSH_MCP_SERVER_VERSION=0.1.0 # Connection Configuration SSH_MCP_CONNECTION_TIMEOUT=30 SSH_MCP_COMMAND_TIMEOUT=60 SSH_MCP_RECONNECT_ATTEMPTS=3 SSH_MCP_RECONNECT_DELAY=5 # Security Configuration SSH_MCP_ALLOW_PASSWORD_AUTH=true SSH_MCP_ALLOW_AGENT_AUTH=true SSH_MCP_ALLOW_KEY_AUTH=true # Logging Configuration SSH_MCP_LOG_FILE= SSH_MCP_LOG_FORMAT=%timestamp% - %name% - %level% - %message%

사용 가능한 도구들

ssh_connect
원격 서버에 SSH 연결을 설정합니다.

필수 매개변수: host, username

선택 매개변수: port, password, privateKey, passphrase, useAgent

ssh_execute_command
원격 서버에서 명령어를 실행합니다.

필수 매개변수: connectionId, command

선택 매개변수: cwd, timeout

ssh_list_connections
현재 활성화된 SSH 연결 목록을 표시합니다.

매개변수: 없음

ssh_disconnect
SSH 연결을 종료합니다.

필수 매개변수: connectionId

ssh_read_file
원격 서버의 파일을 읽습니다.

필수 매개변수: connectionId, path

선택 매개변수: encoding

ssh_write_file
원격 서버에 파일을 씁니다.

필수 매개변수: connectionId, path, content

선택 매개변수: encoding

ssh_list_files
원격 서버의 디렉토리 내용을 나열합니다.

필수 매개변수: connectionId

선택 매개변수: path

사용 예제

1. SSH 연결 설정

// 비밀번호 인증 ssh_connect({ host: "192.168.1.100", username: "admin", password: "your_password" }) // SSH 키 인증 ssh_connect({ host: "192.168.1.100", username: "admin", privateKey: "-----BEGIN PRIVATE KEY-----\n...", passphrase: "key_passphrase" })

2. 명령어 실행

ssh_execute_command({ connectionId: "admin@192.168.1.100:22", command: "ls -la /home/admin", cwd: "/home/admin" })

3. 파일 읽기

ssh_read_file({ connectionId: "admin@192.168.1.100:22", path: "/etc/hosts" })

4. 파일 쓰기

ssh_write_file({ connectionId: "admin@192.168.1.100:22", path: "/tmp/test.txt", content: "Hello, World!" })

🔧 문제해결

일반적인 문제들

연결 실패

SSH 연결이 실패하는 경우 다음을 확인하세요:

  • 호스트 주소와 포트가 올바른지 확인
  • 사용자명과 비밀번호/키가 정확한지 확인
  • 방화벽 설정 확인
  • SSH 서비스가 실행 중인지 확인
인증 실패

인증이 실패하는 경우 다음을 확인하세요:

  • 비밀번호가 정확한지 확인
  • SSH 키 형식이 올바른지 확인 (PEM 형식 필요)
  • 키 패스프레이즈가 정확한지 확인
  • SSH 에이전트가 실행 중인지 확인

🔑 SSH 키 형식 문제 해결

SSH MCP Server는 PEM 형식의 SSH 키만 지원합니다. OpenSSH 형식의 키를 사용하는 경우 다음 명령어로 변환하세요:

ssh-keygen -p -m PEM -f ~/.ssh/id_rsa -N ""

주의: 이 명령어는 기존 키를 PEM 형식으로 변환합니다. 백업을 권장합니다.

명령어 실행 실패

명령어 실행이 실패하는 경우 다음을 확인하세요:

  • 명령어 구문이 올바른지 확인
  • 실행 권한이 있는지 확인
  • 작업 디렉토리가 존재하는지 확인
  • 타임아웃 설정 확인

🔧 라이브러리 호환성 문제

SSH MCP Server는 ssh2 라이브러리를 사용하며, 다음과 같은 호환성 요구사항이 있습니다:

Node.js 버전 호환성

  • 지원 버전: Node.js 18.0.0 이상
  • 권장 버전: Node.js 20.x LTS
  • 확인 방법: node --version

ssh2 라이브러리 관련 문제

다음 오류 메시지가 나타나는 경우:

Error: Cannot parse private key Error: All configured authentication methods failed ECONNRESET: Connection reset by peer

해결 방법:

  1. 의존성 재설치:
    2. rm -rf node_modules package-lock.json npm install
  2. 네이티브 모듈 재빌드:
    3. npm rebuild
  3. Python 환경 확인 (Windows):
    4. npm install --global windows-build-tools
🔐 SSH 키 형식 및 PEM 접속 문제

SSH 키 형식 확인

SSH 키 파일의 첫 줄을 확인하여 형식을 판단할 수 있습니다:

✅ 올바른 PEM 형식

-----BEGIN RSA PRIVATE KEY----- -----BEGIN PRIVATE KEY----- -----BEGIN EC PRIVATE KEY----- -----BEGIN DSA PRIVATE KEY-----

❌ 지원하지 않는 OpenSSH 형식

-----BEGIN OPENSSH PRIVATE KEY-----

SSH 키 변환 방법

OpenSSH 형식을 PEM 형식으로 변환:

1. 백업 생성
cp ~/.ssh/id_rsa ~/.ssh/id_rsa.backup
2. PEM 형식으로 변환
# 패스프레이즈 없는 키 변환 ssh-keygen -p -m PEM -f ~/.ssh/id_rsa -N "" # 패스프레이즈 있는 키 변환 (기존 패스프레이즈 유지) ssh-keygen -p -m PEM -f ~/.ssh/id_rsa
3. 새로운 PEM 키 생성
# RSA 키 생성 (PEM 형식) ssh-keygen -t rsa -b 4096 -m PEM -f ~/.ssh/id_rsa_pem -C "your_email@example.com" # Ed25519 키를 PEM 형식으로 변환 ssh-keygen -p -m PEM -f ~/.ssh/id_ed25519

PEM 키 검증

# 키 형식 확인 head -1 ~/.ssh/id_rsa # 키 유효성 검사 ssh-keygen -l -f ~/.ssh/id_rsa # 공개키 생성 확인 ssh-keygen -y -f ~/.ssh/id_rsa
